# 位置编码（Positional Encoding）深度解析

> 对应公式：原始 Transformer 论文 "Attention Is All You Need"（2017）

---

## 公式

$$PE_{(pos,\ 2i)} = \sin\!\left(\frac{pos}{10000^{2i/d}}\right)$$

$$PE_{(pos,\ 2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d}}\right)$$

---

## 为什么需要位置编码？

Transformer 的 Self-Attention 本质上是**位置无关**的：

```
"猫 吃 鱼"  vs  "鱼 吃 猫"

没有位置信息时，Attention 看到的是同一组 token
完全不知道谁在前谁在后
→ 必须将位置信息注入到每个 token 的表示中
```

---

## 符号含义

| 符号 | 含义 |
|------|------|
| `pos` | token 在序列中的位置（0, 1, 2, ..., T-1） |
| `i` | 嵌入向量中的维度索引（0, 1, ..., d/2-1） |
| `d` | 嵌入维度总大小（如 768） |
| `2i` | 偶数维度 → 用 sin |
| `2i+1` | 奇数维度 → 用 cos |
| `10000` | 超参数，控制频率范围 |

---

## 核心直觉：多频率钟表

每个维度 `i` 对应一个不同频率的正弦波：

```
维度 i=0:   sin(pos / 10000^(0/768))  ≈ sin(pos)        频率最高，波长 ≈ 6
维度 i=1:   sin(pos / 10000^(2/768))  ≈ sin(pos/1.01)   频率稍低
...
维度 i=383: sin(pos / 10000^(766/768)) ≈ sin(pos/9770)  频率极低，波长 ≈ 62832
```

**类比时钟面板：**
```
低维度 = 秒针（转得快，分辨相邻位置）
中维度 = 分针
高维度 = 时针（转得极慢，编码长距离位置）
```

每个位置都获得一个**唯一的"指纹"向量**，无论序列多长。

---

## 为什么用 sin/cos 组合？

**关键性质：任意固定偏移 `k` 的位置关系可用线性变换表示：**

$$PE(pos + k) = M_k \cdot PE(pos)$$

其中 $M_k$ 是一个旋转矩阵：

$$\begin{bmatrix}\sin(pos+k)\\\cos(pos+k)\end{bmatrix}
= \begin{bmatrix}\cos k & \sin k\\-\sin k & \cos k\end{bmatrix}
\begin{bmatrix}\sin(pos)\\\cos(pos)\end{bmatrix}$$

**含义：**
- `pos+k` 的编码 = 旋转矩阵 × `pos` 的编码
- 模型可以轻松学到"向右移 k 步"这种相对位置关系
- 不需要记住每个绝对位置

---

## 为什么底数是 10000？

```
频率 = 1 / 10000^(2i/d)

i=0:   频率 = 1         → 波长 2π ≈ 6      （区分相邻位置）
i=d/2: 频率 = 1/10000   → 波长 ≈ 62832    （覆盖极远距离）
```

10000 让频率从 **1 到 1/10000** 呈指数递减：
- 低维度：高频 → 精确区分近处位置
- 高维度：低频 → 粗略编码远处位置

**类比二进制：**
```
位置 0: 0000
位置 1: 0001  ← 最低位（最高频）变化最快
位置 2: 0010
位置 4: 0100  ← 高位（低频）变化慢
```

sin/cos 位置编码 = **连续版的二进制**

---

## 具体数字示例

假设 `d = 4`，计算位置 `pos = 3` 的编码：

```
i=0: PE(3, 0) = sin(3 / 10000^(0/4)) = sin(3)    ≈  0.14
     PE(3, 1) = cos(3 / 10000^(0/4)) = cos(3)    ≈ -0.99

i=1: PE(3, 2) = sin(3 / 10000^(2/4)) = sin(0.03) ≈  0.03
     PE(3, 3) = cos(3 / 10000^(2/4)) = cos(0.03) ≈  1.00

位置 3 的编码向量 = [0.14, -0.99, 0.03, 1.00]
```

## 三种位置编码方式对比

| 方式 | 原始 Transformer | nanoGPT (GPT-2) | nanochat (RoPE) |
|------|-----------------|-----------------|-----------------|
| 方法 | sin/cos 固定公式 | 可学习 Embedding | 旋转位置编码 |
| 是否需要学习 | ❌ 无参数 | ✅ 需要学习 | ❌ 无参数 |
| 长度外推 | ✅ 理论上无限 | ❌ 最多 block_size | ✅ 支持外推 |
| 参数量 | 0 | block_size × n_embd | 0 |
| 现代 LLM 使用 | 较少 | GPT-2 系列 | LLaMA/Mistral/nanochat |

---

## RoPE 与 sin/cos PE 的关系

RoPE（Rotary Position Embedding）是 sin/cos PE 的进化版：

```
sin/cos PE：
  位置信息"加"到 token 向量上（x + PE）
  相对位置关系是隐式的

RoPE：
  直接把位置信息"旋转"到 Q、K 向量上
  Q 和 K 做点积时，相对位置差自然体现出来
  → 注意力分数天然包含相对位置信息
  → 支持更长的序列外推
```

---

## nanoGPT 的实现

nanoGPT 使用**可学习的位置 Embedding**，而非公式：

```python
# 初始化
wpe = nn.Embedding(block_size, n_embd)  # 1024 × 768

# 前向传播
pos = torch.arange(0, T, dtype=torch.long)
pos_emb = self.transformer.wpe(pos)     # [T, C]
x = tok_emb + pos_emb                   # 直接相加
```

优点：让模型自己学最适合的位置表示
缺点：无法处理超过 `block_size=1024` 的序列
