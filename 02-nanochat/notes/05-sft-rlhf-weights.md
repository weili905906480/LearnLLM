# SFT / RLHF / DPO 改变了哪些权重？

> 核心问题：三个训练阶段分别"改变"了模型的哪些部分？

---

## 三个阶段的关系

```
预训练              SFT                RLHF / DPO
  │                  │                    │
  ▼                  ▼                    ▼
学会语言           学会对话             学会"讨人喜欢"
（知道什么）       （怎么回答）         （怎么回答得更好）

改动幅度：大        中                   小
数据类型：文本      指令+回复            好回复 vs 差回复的对比
```

---

## 先理解 QKV 的本质

```
Q = x @ W_Q    "我想找什么信息？"  ← Query（查询）
K = x @ W_K    "我有什么信息？"    ← Key（索引）
V = x @ W_V    "我实际传出什么？"  ← Value（内容）
```

**图书馆比喻：**
```
Q = 你脑子里的问题："找关于深度学习的书"
K = 每本书的标签："本书内容：深度学习/Python"
V = 书的实际内容

W_Q、W_K 决定"关注什么"
W_V 决定"传递什么"
```

**W_Q 和 W_K 控制的是注意力模式——谁关注谁。**

## SFT 改变了什么？

### 行为变化

```
预训练后（续写模式）：
  问："法国的首都是哪里？"
  答："法国的首都是哪里？这个问题很有趣……"（继续续写）

SFT 后（对话模式）：
  问："法国的首都是哪里？"
  答："法国的首都是巴黎。"（识别指令，直接回答）
```

### 权重层面的变化

**W_Q 的变化（最大）：**
```
预训练：Q 主要响应"上文语境词汇"
SFT 后：Q 增加了对"指令意图词"的响应
        （"请"、"告诉我"、"解释"→ 触发回答模式）
```

**W_K 的变化：**
```
预训练：普通续写 token 的可索引性均等
SFT 后："指令相关 token"变得更容易被 Q 检索到
```

**W_V 的变化：**
```
预训练：传递丰富的上文语义（适合续写）
SFT 后：强化"直接回答"相关信息的传递权重
```

**W_MLP 的变化（最小）：**
```
"巴黎是法国首都"这个知识已经存在 MLP 里
SFT 不需要重写知识，只需改变"找到并表达"的方式
→ MLP 几乎不动
```

---

## 为什么 MLP 存知识，QKV 控行为？

### MLP 是键值存储

```python
# MLP 工作机制
x = Linear(x)    # 激活"知识神经元"（Key）
x = GELU(x)      # 稀疏激活（只有相关神经元响应）
x = Linear(x)    # 读取对应"知识内容"（Value）

# "法国的首都是巴黎"就存在某个神经元的权重里
```

### QKV 控制信息流动

```
SFT 的目标：改变"说话方式"，不是"学新知识"

类比：
  你已经知道"巴黎是法国首都"（MLP 里有）
  SFT = 培训你用"法国的首都是巴黎"来回答
        不是让你重学地理
  → 只需改 QKV 的注意力模式，不需要改 MLP
```

## RLHF 的权重变化

### 两个模型

```
Policy Model  ← 被训练，权重会更新
Reference Model ← SFT 模型的拷贝，永远冻结！
（+ Reward Model：单独训练的打分模型）
```

### KL 惩罚机制

```python
# PPO 优化目标
objective = reward - β * KL(policy || reference)

# reward：Reward Model 的打分
# KL：Policy 与 Reference 的分布距离
# β：控制"自由度"
```

```
没有 KL 惩罚 → 奖励黑客：
  模型学会输出"这是个好问题！我完全同意！"
  （讨好用户，reward 高但无用）

有 KL 惩罚 → 权重不能偏离 Reference 太远
  → 既获得高奖励，又保留 SFT 的能力
```

### RLHF 对各层的影响

```
深层 Attention QKV（改动最大）：
  W_Q 新增对"危险/安全意图"的识别
  W_K 调整"安全相关 token"的可检索性
  W_V 强化"拒绝/有帮助回复"的输出

深层 MLP（较小改动）：
  知识尽量保留，只微调输出倾向

lm_head（轻微）：
  调整词汇偏好（如更倾向生成礼貌用语）
```

## DPO 的权重变化

### 核心简化

```
RLHF：prompt → Policy → Reward Model 打分 → PPO 更新
DPO： 直接用 (prompt, chosen, rejected) 对训练

绕开 Reward Model 和 PPO！
```

### DPO Loss

```python
loss = -log(sigmoid(
    β * (log π(chosen|x)   - log π_ref(chosen|x))
   -β * (log π(rejected|x) - log π_ref(rejected|x))
))
```

**直觉：**
```
对 chosen  回复：让 Policy 比 Reference 更倾向生成它  ↑
对 rejected 回复：让 Policy 比 Reference 更不倾向生成它 ↓
Reference 作为"锚点"，防止权重走偏太远
```

## 三阶段权重影响对比

```
┌──────────────┬──────────┬──────────────────────────────────┐
│ 阶段         │ 改动幅度 │ 改动方向                          │
├──────────────┼──────────┼──────────────────────────────────┤
│ 预训练       │ ████████ │ 随机 → 会语言、有知识             │
├──────────────┼──────────┼──────────────────────────────────┤
│ SFT          │ ███░░░░░ │ 续写模式 → 对话模式               │
│              │          │ 主要改：深层 QKV                  │
│              │          │ 几乎不动：MLP                     │
├──────────────┼──────────┼──────────────────────────────────┤
│ RLHF / DPO   │ █░░░░░░░ │ 对话模式 → 人类偏好对话模式        │
│              │          │ 主要改：深层 QKV                  │
│              │          │ 极少改动：MLP                     │
└──────────────┴──────────┴──────────────────────────────────┘
```

### 各层改动程度

| 权重位置 | SFT | RLHF/DPO | 原因 |
|---------|-----|----------|------|
| 深层 Attention QKV | ████ | ████ | 行为模式在这里改变 |
| 深层 MLP | ██ | █ | 知识库尽量保留 |
| 浅层 Attention/MLP | ██ | █ | 基础语言能力已有 |
| lm_head | █ | █ | 词汇偏好微调 |
| Embedding (wte/wpe) | █ | ░ | 词汇语义基本稳定 |

## LoRA：只改部分权重

```
Full SFT（全量微调）：
  所有权重可更新
  需要和预训练一样大的显存

LoRA（低秩适配）：
  冻结原始权重 W
  只训练两个小矩阵 A（d×r）和 B（r×d）
  ΔW = A × B，其中 r 很小（如 8 或 16）

  输入 x → W·x + (A·B)·x → 输出
           ↑原始权重     ↑LoRA新增

优点：显存需求极低，效果接近全量
```

**LoRA 通常只加在 Q、V 投影矩阵上**（不加在 K 上），因为研究发现 K 对行为的影响相对较小。

---

## 一句话总结

> **三个阶段都修改全部权重，但幅度递减（预训练>SFT>RLHF）；本质上改变的不是"知识"（存在 MLP），而是"信息流动方式"（QKV 控制）——SFT 改变说话方式，RLHF/DPO 改变偏好对齐，主要体现在深层 Attention 的 QKV 权重上。**
