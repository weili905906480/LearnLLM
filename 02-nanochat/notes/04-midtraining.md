# Midtraining（中期训练）

> nanochat Stage 3：在高质量数据上继续训练

## 目标

在基础预训练之后，用**更高质量、更有针对性**的数据继续训练，提升模型的知识密度和能力。

## 为什么需要 Midtraining？

```
预训练数据（海量但质量参差不齐）
        ↓ Stage 2: 学会语言 + 粗略知识
        
中期训练数据（少量但高质量）
        ↓ Stage 3: 加深知识 + 提升推理
        
SFT 数据（对话格式）
        ↓ Stage 4: 学会聊天
```

**类比：**
- 预训练 = 广泛阅读（报纸、小说、网页...什么都看）
- Midtraining = 精读教材（教科书、百科全书、高质量文章）

### 2. 与预训练的区别

| 对比 | Pretraining | Midtraining |
|------|-------------|-------------|
| 数据量 | 海量（B级 tokens） | 较少（精选） |
| 数据质量 | 混合 | 高质量 |
| 学习率 | 正常 schedule | 通常更小 |
| 目标 | 学语言 + 广泛知识 | 深化知识 + 能力 |
| 训练方式 | 同样是 next-token prediction | 同上 |
