# Reinforcement Learning（强化学习对齐）

> nanochat Stage 5（可选）：进一步对齐模型行为

## 目标

通过强化学习让模型的输出更符合人类偏好：更有帮助、更安全、更诚实。

## 为什么 SFT 还不够？

```
SFT 的局限：
- 只能模仿训练数据中的"好回答"
- 无法表达"A 比 B 好"这种偏好信号
- 对边界情况（有害内容等）处理不够好

RL 的优势：
- 可以用 reward signal 指导模型
- 能处理"程度"问题（不只是对/错）
- 能在 SFT 基础上进一步提升质量
```

## 主流方法

### 1. RLHF（Reinforcement Learning from Human Feedback）

```
流程：
1. 收集人类偏好数据：给同一个 prompt 的两个回复排序
2. 训练 Reward Model：学习人类偏好
3. 用 PPO 算法优化：让模型输出获得更高 reward

       Human Preference Data
              ↓
       Reward Model (RM)
              ↓
       PPO Training
              ↓
       Aligned Model
```

- [ ] Reward Model 的训练
- [ ] PPO 算法的基本思想
- [ ] KL 散度惩罚（防止偏离太远）

### 2. DPO（Direct Preference Optimization）

```
简化思路：
- 跳过 Reward Model
- 直接从偏好数据优化模型
- 数学上等价于 RLHF，但实现更简单

       Human Preference Data
              ↓
       DPO Loss 直接优化
              ↓
       Aligned Model
```

- [ ] DPO 的核心公式直觉
- [ ] 相比 RLHF 的优缺点
- [ ] 为什么 DPO 实现更简单
